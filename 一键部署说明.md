---

## 详细步骤说明

### 第一次部署

脚本会自动帮你完成：

#### ✅ 自动检查和安装
- 检查 Git 是否安装（如未安装会打开下载页面）
- 配置 GitHub 账号信息
- 初始化 Git 仓库

#### ✅ 自动创建仓库
- 打开 GitHub 仓库创建页面
- 你只需要：
  1. 输入仓库名：`research-task-system`
  2. 选择 **Public**（公开）
  3. 点击 **Create repository**
  4. 复制仓库地址，粘贴回脚本

#### ✅ 自动推送代码
- 自动提交所有文件
- 推送到 GitHub
- 如需身份验证，会自动打开 Token 创建页面

#### ✅ 自动配置部署
- 打开 GitHub Pages 设置页面
- 你只需要：
  1. 选择 **main** 分支
  2. 点击 **Save**

### 完成后

你会得到一个网址，例如：
```
https://你的用户名.github.io/research-task-system/
```

**分享这个网址给团队成员，他们就可以：**
- 📝 注册账号
- 🔐 登录系统
- ✅ 管理任务
- 📊 查看进度

## 常见问题

### Q1: 没有 GitHub 账号怎么办？
**A:** 脚本会询问你，并自动打开注册页面。注册是免费的！

### Q2: 不知道 Git 是什么？
**A:** 不用担心！脚本会检测，如果没安装会自动打开下载页面，安装很简单。

### Q3: 推送时要求输入密码？
**A:** GitHub 现在使用 Token 代替密码。脚本会自动打开 Token 创建页面，复制生成的 Token 作为密码即可。

### Q4: 部署后无法访问？
**A:** 等待 1-2 分钟让 GitHub 完成部署。如果还是不行，检查 GitHub Pages 设置是否正确。

### Q5: 如何更新系统？
**A:** 修改文件后，再次运行 `一键部署.bat` 即可！
